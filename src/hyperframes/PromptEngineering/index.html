01
Technique 01
Zero-Shot
Ask directly. No context, no examples, no guidance.
what you type
"My Python code is slow.
How do I make it faster?"
what you get
Start by profiling with cProfile or line_profiler to identify bottlenecks before optimizing blindly. Focus on improving algorithmic complexity first — switching from O(n²) to O(n log n) yields the biggest gains. Leverage Python's built-in functions and standard library modules, which are implemented in C.
Generic. Shallow. Could be anyone.
02
Technique 02
Role Prompting
Give the AI a persona. Define who it is.
what you type
You are a senior Python performance engineer
with 10 years of experience optimising
production systems at scale.
My Python code is slow. How do I make it faster?
what you get
Start by profiling with cProfile — never optimize blind. Once you've pinpointed hotspots, analyze algorithmic complexity; an O(n²) loop won't be fixed by micro-optimization. Offload numerical ops to NumPy or use C-backed builtins like map() and itertools.
Expert. Targeted. Professional.
03
Technique 03
Few-Shot Prompting
Show examples. The AI learns your pattern.
what you type
Q: My React app re-renders too much. Fix? → Use React.memo, move state closer to usage Q: My SQL queries time out. Fix? → Add indexes on WHERE/JOIN columns, use EXPLAIN Q: My Python code is slow. How do I make it faster? A:
what you get
Profile with cProfile, replace loops with NumPy vectorization, and cache repeated computations with functools.lru_cache.
Pattern-matched. Precise. Consistent.
04
Technique 04
Chain-of-Thought
Three words that unlock step-by-step reasoning.
what you type
My Python code is slow. How do I make it faster?
Think step by step before giving your final recommendation.
what you get
Step 1 Diagnose — what kind of slow? I/O-bound, CPU-bound, or memory-bound?
Step 2 Profile with cProfile or line_profiler to pinpoint the exact bottleneck.
Step 3 Fix the hotspot — vectorize with NumPy, cache with lru_cache, use generators.
Final Never guess — profile first, then optimize the 20% causing 80% of slowness.
Structured. Deep. Reasoned.
05
Technique 05
Self-Consistency
Ask the same question multiple ways. Vote on the best answer.
what you type (3 variations)
"My Python code is slow. How do I make it faster?" "Performance issues with my Python app. What optimizations?" "Python script too slow in production. Most impactful ways to speed it up?"
consensus (what you get)
All 3 paths agree:

Profile / diagnose first — always find the actual bottleneck before optimizing anything.

Replace pure Python loops with NumPy, async, or compiled code once confirmed.
Reliable. Consensus-driven.
06
Technique 06
Tree-of-Thoughts
Explore multiple strategies simultaneously. Synthesise the best.
what you type
"My Python code is slow.
How do I make it faster?"
what you get
Approach 1 — Algorithmic
Fix O(n²) loops → O(n log n). Use sets and dicts over lists for lookups.
Approach 2 — Language-level
Replace loops with NumPy vectorization. Use generators and lru_cache to cut overhead.
Approach 3 — Infrastructure
Multiprocessing for CPU, async for I/O, Cython for hot paths.
★ synthesis
Profile first. Fix algorithms (biggest win), then vectorize, then parallelize only if needed.
Comprehensive. Best combined.
07
Technique 07
ReAct
Reason, then act, then observe. Loop until solved.
what you type
"My Python code is slow.
How do I make it faster?"
AI uses: Thought → Action → Observation loop
what you get
Thought I need to identify the type of slowness before recommending fixes.
Action Run cProfile to find the top 5 slowest functions.
Observation Bottleneck is a nested loop doing repeated list membership checks.
Thought List lookups are O(n) — converting to a set gives O(1).
Final Answer Profile with cProfile. Convert list lookups to sets. Vectorize remaining loops with NumPy.
Evidence-based. Self-correcting.
