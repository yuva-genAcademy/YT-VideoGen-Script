How The Model Sees Language
Tokens, Not Characters.
tokens
The model att ends to tokens
The transformer attends to tokens — not to words, syllables, or characters.
no native sense of spelling
"strawberry" → straw berry
The model doesn't see individual letters.
why models fail at character tasks

"How many letter 'r's are in 'strawberry'?"

Not because it's unintelligent —
but because tokens don't preserve character-level information.
straw + berry — the model cannot introspect individual letters.
Design Principles
What This Means in Practice
Rule 01 — Compress
Shorter prompts.
Same information.
Reduces cost and latency. Prompt compression is a real skill.
Rule 02 — Format Costs
JSON > plain text.
XML > plain text.
Same data — structured formats cost more tokens.
Rule 03 — Budget by Language
Non-Latin scripts
cost more.
Multilingual apps: allocate extra budget for Thai, Arabic, CJK.
Rule 04 — Track Context
You have to
track it.
The model cannot signal it's losing context. You must track token count.
When debugging unexpected behavior — forgotten instructions, incomplete reasoning — check your token count first.
