Phase 1 — Pre-Training
Architecture
Transformer
Attention mechanisms learn which tokens in a sequence
are relevant to each other — regardless of distance.
Every major LLM today:
GPT
Claude
Gemini
LLaMA
All transformers.
Training Data
Trillions
of tokens
Web crawls · books · code · scientific papers
All of English Wikipedia ≈ four billion tokens.
A rounding error at this scale.
The Training Task
Predict the
next token.
Solved across all of human writing — grammar, logic, facts, and code all help prediction.
How it scales
Self-supervised learning
No human labels required.
The label is always the next word — already in the text.
That is why pre-training can scale so aggressively.
Result
Base Model
Powerful text predictor.
No goal. No safety properties.
Not a product — a starting point.
The Compute Cost
$10M – $100M+
Months of GPU time. This is the moat separating
a handful of labs from everyone else.
Phase 2 — Instruction Tuning & RLHF
Making the base model useful
Instruction Tuning
Training on examples of human instructions paired with high-quality responses.
The model learns to follow directives — not just complete text.
Critical Variant
RLHF — Reinforcement Learning from Human Feedback
1
Human raters compare two outputs and pick the better one
Repeat at scale.
2
A reward model learns to predict those preferences
A separate model trained on the human ratings.
3
The main model is optimized to maximize that reward
Reinforcement learning shapes the model toward preferred outputs.
Why it mattered
ChatGPT felt different.
Not just more capable — shaped toward outputs humans actually prefer. A different optimization target than raw prediction.
Key Constraint
Behavior, not knowledge.
Fine-tuning shapes how the model responds.
To give it new knowledge: retrain it —
or inject it at inference time via RAG.
Phase 3 — Alignment
Makes the model safe and honest.
Layer 1
Hard Content Filtering
Rule-based guardrails — certain categories blocked regardless of framing.
Layer 2
Preference-based Tuning
Teaches the model to be helpful without being harmful.
Anthropic's Approach
Constitutional AI
The model critiques and revises its own outputs against a written set of principles. Less dependence on continuous human feedback.
Practical Tradeoff
Over-refusal
Declining legitimate requests because they superficially resemble harmful ones. Knowing this shapes your prompt strategy.
The Knowledge Cutoff
Pre-training happens once on a static dataset. Every model has a date beyond which it has no information. Anything after that date does not exist to the model — unless you tell it.
CUTOFF
Training data Unknown →
Why models hallucinate confidently
Volume ≠ Ground Truth
The model's certainty reflects how often something appears in the corpus — not whether it is actually true.
For Builders
RAG — supplement frozen knowledge with live retrieval
Tell the model today's date — it shifts behavior meaningfully
Recent cutoff ≠ better model — training quality matters more

And underneath all of it — a transformer, learning from patterns in text at a scale that is genuinely hard to reason about intuitively.
When a model hallucinates, refuses, or is inconsistent — the root cause lives in one of these phases. That diagnostic ability separates engineers who understand the stack from those who are just prompting and hoping.
